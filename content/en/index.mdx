A modular, extensible WeChat iLink Bot SDK — available in **Node.js**, **Python**, **Go**, and **Rust**. Plus a **Pi coding agent extension** that lets you chat with Pi from WeChat. Cross-platform: tested on **Linux, macOS, and Windows**. [GitHub](https://github.com/corespeed-io/wechatbot). [![CI](https://github.com/corespeed-io/wechatbot/actions/workflows/ci.yml/badge.svg)](https://github.com/corespeed-io/wechatbot/actions/workflows/ci.yml)

<DemoVideo src="/demo.mp4" caption="Scan QR in WeChat → Chat with Pi coding agent" />

<Section id="quick-start" title="Quick start">

```bash
npm install @wechatbot/wechatbot
```

```typescript
import { WeChatBot } from '@wechatbot/wechatbot'

const bot = new WeChatBot()
await bot.login()              // Shows QR code → scan in WeChat

bot.onMessage(async (msg) => {
  await bot.sendTyping(msg.userId)
  await bot.reply(msg, `Echo: ${msg.text}`)
})

await bot.start()
```

Scan the QR code in WeChat → your bot is live. Credentials auto-saved, session auto-recovered.

<Section id="architecture" title="Architecture">

All SDKs follow the same layered design:

<CodeBlock lang="mermaid" lineHeight="1.3" showLineNumbers={false}>
{`graph TD
    A["🤖 Your Bot Code"] --> B["Bot Client (Orchestrator)"]
    B --> C["Poller"]
    B --> D["Sender"]
    B --> E["Typing"]
    B --> F["Media"]
    C --> G["Context Store (Token Cache)"]
    D --> G
    E --> G
    F --> G
    G --> H["Protocol / API (HTTP Calls)"]
    H --> I["Storage (Credentials + State)"]`}
</CodeBlock>

Shared capabilities: QR login, long-poll, context_token management, AES-128-ECB CDN crypto, session recovery, smart text chunking.
